
# Phased Development Plan — ✅ Completed

## 1. Project Context & Strict Constraints

This is a critical bug-fix and enhancement cycle for Somnus v1.5. The objective is to resolve fatal `EncodingError` crashes during `Tone.loaded()` caused by 404 HTML fallbacks, fix dead audio parameters caused by improper Tone.js timing scheduling within the automation loop, and implement strict `localStorage` data persistence for all user settings to survive page reloads.

## 2. Enforced Dependencies & Architecture

* **Audio Engine:** Tone.js (v15.0.4). All `rampTo` calls executed inside `scheduleRepeat` MUST accept and pass the atomic `time` parameter to prevent scheduling conflicts.
* **Error Boundaries:** The `Tone.loaded()` await block must be wrapped in a `try/catch` to prevent local dev server 404s (which return HTML) from causing fatal decoding errors that freeze the entire engine.
* **State Management:** React `AppContext` must serialize and deserialize `sessionDuration`, `currentState`, `solTarget`, and `mixerState` via the browser `localStorage` API.
* **TDD Enforcement:** The coding agent is **STRICTLY PROHIBITED** from altering the provided test code. The agent must copy the literal Javascript blocks into the respective test files and write the implementation to make them pass.

### Phase 1: Robust Audio Decoding & Error Boundaries — ✅ Completed

* **Objective:** Prevent missing `.wav` files or Vite 404 fallbacks from halting the entire audio engine boot sequence.
* **Implementation Directives:**
* In `GenerativeAudioController.js` -> `bootEngine()`, wrap `await Tone.loaded()` in a `try/catch` block.
* If it catches an error, log a warning, manually set `this.rainPlayer.loaded = false` and `this.wavesPlayer.loaded = false` (to prevent them from starting later), and forcefully set `this.isInitialized = true` so the other 3 channels (Binaural, Isochronic, Masking) can still function.

describe('Phase 1: Decoding Error Boundaries', () => {
  it('should catch decoding errors and still initialize the engine', async () => {
    // Force Tone.loaded to throw a mock EncodingError

### Phase 2: Tone.js Accurate Timing (Ramp Fix) — ✅ Completed

* **Implementation Directives:**
* Modify `processAutomationTick(payload = {}, elapsedSeconds = 0, time)` to accept the third parameter `time`.
* Update the `scheduleRepeat` callback in `startSession` to pass the time: `this.processAutomationTick(this.currentPayload, Tone.Transport.seconds, time);`.
* Inside `processAutomationTick`, update EVERY `.rampTo(target, duration)` call to be `.rampTo(target, duration, time)`. This includes `masterLpf`, `ch1OscLeft`, `ch1OscRight`, and `ch2Tremolo`.

import React from 'react';

        <TestComponent />
      </AppProvider>
    );

      getByText('Update').click();
    });

    expect(localStorage.getItem('somnus_currentState')).toBe('0.75');
  });
});
