# Phased Development Plan — ✅ ✅

## 1. Project Context & Strict Constraints

Somnus is a mobile web application designed to provide a non-pharmacological auditory intervention for sleep induction and maintenance. The application must be accessed directly through a mobile web browser, eliminating the need for native app downloads. Using a generative music engine, it personalizes an audio stream to smoothly guide users from cognitive hyperarousal into Slow-Wave Sleep (SWS). It achieves this by stabilizing the cortical environment without introducing structural or dynamic distractions.

The core constraints dictate that the app must actively mitigate hyperarousal using consonant intervals, enforce sympatho-respiratory coupling by decelerating the audio tempo from up to 80 BPM down to 50 BPM, and utilize stochastic predictability to close the thalamic gate. Continuous broadband exposure is strictly prohibited to prevent REM suppression.

## 2. Enforced Dependencies & Architecture

* **Frontend Framework:** React (latest stable version) built as a Single-Page Application (SPA).
* **Styling:** Tailwind CSS.
* **Audio Engine:** `Tone.js` paired with the standard Web Audio API `AudioWorklet`.
* **Hosting:** Static site deployment via GitHub Pages.
* **State Management:** Centralized React Context or Zustand.
* **External Assets:** `MIT_KEMAR` impulse response file and Open Binaural Renderer (OBR) WebAssembly (`.wasm`) binaries (to be stored in `public/assets/`).
* **Prohibited Tech:** No Progressive Web App (PWA) manifest or offline service workers for V1. No rigid tempo grid for note generation.
