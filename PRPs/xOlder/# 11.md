# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The native AudioWorklet connection is now successfully processing the immediate-fire Layer 1 drone without throwing errors. However, Layers 2 and 3 remain silent. Code analysis reveals a state mismatch on the `Tone.Transport` timeline: `Tone.Transport.cancel(0)` is wiping scheduled loop events, but conditional creation blocks (`if (!this.harmonicLoop)`) prevent the loops from being re-scheduled on subsequent executions or Vite HMR reloads. 

**Objective:** Harden the Transport and Loop lifecycle within `GenerativeAudioController.js`. We will explicitly dispose of old loop instances, recreate them fresh on every session start, and force a Transport sync offset to guarantee the Web Worker clock picks them up.

## 2. Technical Decisions & Dependencies
* **Lifecycle Hardening:** Replace conditional `Tone.Loop` instantiation with an explicit `dispose()` and recreation pattern.
* **Transport Sync Offset:** Start the transport with `Tone.Transport.start("+0.1")` to give the native Web Worker clock a 100ms buffer to align its internal timeline with the newly injected hardware context.
