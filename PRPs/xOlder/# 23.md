
# Phased Development Plan — [Status Marker] ✅

## 1. Project Context & Strict Constraints

This is a critical architectural repair cycle for Somnus v1.5. The objective is to permanently resolve the "Full Blast" boot issue by strictly welding the React `localStorage` hydration pipeline to the Tone.js singleton, enforcing a mandatory `Tone.start()` user gesture before processing audio, and fixing a catastrophic Tone.js API anti-pattern where synchronous `.value` assignments were destroying scheduled `rampTo` automation curves.

## 2. Enforced Dependencies & Architecture

* **React Hydration Bridge:** `App.jsx` MUST invoke `audioController.updatePayload(payload)` inside its `handleParamUpdate` wrapper to guarantee the Tone.js engine holds the true state.
* **Tone.js Automation Integrity:** In `GenerativeAudioController.js`, ALL instances of `.value = xyz` that occur *immediately after* a `.rampTo()` call must be DELETED. Mixing scheduled automation with synchronous value overriding corrupts the Web Audio API clock and freezes the audio parameters.
* **Tone.js Context Unlock:** `PlayButton.jsx` MUST explicitly `await Tone.start()` *before* invoking `audioController.startSession()`.
* **TDD Enforcement:** The coding agent is **STRICTLY PROHIBITED** from altering the provided test code. The agent must copy the literal Javascript blocks into the respective test files and write the implementation to make them pass.

* **Implementation Directives:**

* `this.ch1OscLeft.frequency.value = leftFreq;`
