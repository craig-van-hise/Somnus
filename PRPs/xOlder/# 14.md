# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The native context injection successfully resolved the `InvalidAccessError` allowing the audio graph to route completely through the `AudioWorkletNode` (proven by Layer 1 drone playing). However, Layers 2 and 3 (Tone.Loop sequences) remain silent. Research into Tone.js (Issue #701) reveals that the static `Tone.Transport` singleton does not update its reference when `Tone.setContext()` is called. Consequently, loop schedulers bind to the new context, but start/stop commands are being sent to a deprecated ghost context.

**Objective:** Refactor `GenerativeAudioController.js` to use dynamic context getters. Replacing `Tone.Transport` with `Tone.getTransport()` will synchronize the timeline commands with the natively injected hardware context, restoring sequence playback.

## 2. Technical Decisions & Dependencies
* **Dynamic Getters:** Strict enforcement of `Tone.getTransport()` over the static `Tone.Transport` reference for all timeline operations (`cancel`, `stop`, `start`, `bpm`, `position`).
* **Preserve Injection:** The module-level `Tone.setContext(nativeCtx)` must remain intact as it is currently holding the Worklet connection stable.
