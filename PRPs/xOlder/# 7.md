# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
Based on the failure report and the source files provided, the `InvalidAccessError` is being caused by an asynchronous race condition mixed with cross-context drift. `Tone.start()` initializes or resumes a context, but `GenerativeAudioController` creates its Tone nodes synchronously *before* awaiting the Worklet registration. When the Worklet eventually instantiates, it occasionally grabs a slightly different context wrapper or executes after Vite HMR has orphaned the original context, severing the graph. 

**Objective:** Permanently resolve the cross-context contamination by flattening the audio graph instantiation into a single, sequential, strictly-awaited asynchronous pipeline (`bootEngine`). We will force the Worklet to load *before* any Tone.js routing is established, ensuring 100% memory space alignment.

## 2. Technical Decisions & Dependencies
* **Architecture Shift (Linear Graph Construction):** We will deprecate the separated `initialize()` and `setupReverbWorklet()` methods. They will be merged into a single `async bootEngine()` method.
* **Context Extraction Standard:** Tone.js context will be extracted once, globally, via `Tone.getContext().rawContext` immediately after `Tone.start()` is triggered by the user.
* **Node Bridging:** The routing will explicitly follow this chain: `Tone.PolySynth` -> `Tone.Gain (reverbInputBus)` -> `Tone.connect(reverbInputBus, nativeWorklet)` -> `nativeGain` -> `nativeContext.destination`.

        ```javascript
        async bootEngine() {
          if (this.isInitialized) return;

          // 1. Extract unified native context
          const nativeContext = Tone.getContext().rawContext;

          // 2. Await Worklet module registration BEFORE routing
          try {
            await nativeContext.audioWorklet.addModule('/assets/fdn-reverb-worklet.js');
            this.reverbWorklet = new AudioWorkletNode(nativeContext, 'fdn-reverb-worklet', {
              numberOfInputs: 1,
              numberOfOutputs: 1,
              outputChannelCount: [2],
            });
            this.reverbNode = this.reverbWorklet;
          } catch (e) {
            console.warn('Worklet load fallback/test environment:', e);
            this.reverbWorklet = null;
          }

          // 3. Instantiate Tone nodes
          this.masterBus = new Tone.Volume(-6).toDestination();
          this.reverbInputBus = new Tone.Gain(1);

          // 4. Implement Native Bridge Routing
          if (this.reverbWorklet) {
            // Bridge Tone -> Native
            Tone.connect(this.reverbInputBus, this.reverbWorklet);
            
            // Bridge Native -> Hardware Destination
            const nativeGain = nativeContext.createGain();
            nativeGain.gain.value = 0.5; // -6dB headroom
            this.reverbWorklet.connect(nativeGain);
            nativeGain.connect(nativeContext.destination);
          } else {
            // Fallback if Worklet fails (e.g., tests)
            this.reverbInputBus.connect(this.masterBus);
          }

          // 5. Re-instantiate Layer 1, 2, and 3 Synths precisely as they were 
          // in the original initialize() method, connecting them to this.reverbInputBus.
          // (Copy over your original synth initialization logic here)
          
          this.isInitialized = true;
        }
        ```
    4.  Update `async startSession()` to call `await this.bootEngine()` at the very beginning, removing calls to `initialize()` and `setupReverbWorklet()`.
