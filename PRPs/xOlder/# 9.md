# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The audio graph is currently failing with an `InvalidAccessError` at the boundary between Tone.js nodes and the native `AudioWorkletNode`. This occurs because Tone.js (v15) automatically wraps its default context in a `standardized-audio-context` polyfill. Bridging a wrapped Tone node to a strictly native Worklet node triggers cross-context contamination rules within the browser. 

**Objective:** Bypass the Tone.js context polyfill entirely. We will inject a pure native `BaseAudioContext` into Tone.js upon user interaction, guaranteeing that all Tone nodes and the Worklet node instantiate on the exact same underlying hardware context pointer.

## 2. Technical Decisions & Dependencies
* **Context Injection:** We will instantiate `new window.AudioContext()` inside `PlayButton.jsx` and bind it via `Tone.setContext()` before `Tone.start()` executes.
* **Test Environment Resilience:** The injection must verify the existence of `window.AudioContext` to avoid breaking the JSDOM Vitest environment.
* **Extraction Reversion:** With the polyfill bypassed, `GenerativeAudioController.js` can revert to standard `Tone.getContext().rawContext` extraction without throwing type errors.

          }

          }
