

# Product Requirements Prompt (PRP) — [Status Marker] ✅

## 1. Project Context & Objectives

The application successfully generates audio, but iOS Safari suspends the Web Audio API context when the device screen locks. The existing `silence.wav` HTML5 keep-alive mechanism fails to initialize because it is invoked asynchronously via `setTimeout`, violating iOS WebKit's strict user-gesture requirements. This PRP refactors the engine to trigger the HTML5 background audio element synchronously at the exact moment the user taps the Play button.

## 2. Technical Decisions & Dependencies

* **Synchronous Invocation:** We will expose `audioController.startIOSKeepAlive()` and `audioController.setupMediaSession()` so they can be called synchronously in `PlayButton.jsx` before any `await` operators yield the main thread.
* **Idempotent Execution:** `startIOSKeepAlive()` must be refactored to be safe to call multiple times without spawning duplicate `<audio>` DOM elements.

## 3. Implementation Phases

### Phase 1: Refactor GenerativeAudioController Keep-Alive — [Status Marker] ✅

* **Objective:** Prepare the keep-alive and MediaSession methods to be called synchronously, removing deferred execution.
* **Instructions:**
1. **Update `startIOSKeepAlive()**`: Modify the function in `src/engine/GenerativeAudioController.js` to reuse the existing `<audio>` element if it exists, rather than exiting early.
```javascript
startIOSKeepAlive() {
  try {
    if (typeof document === 'undefined') return;

    if (!this.iosKeepAliveAudio) {
      const audio = document.createElement('audio');
      audio.src = `${BASE}assets/silence.wav`;
      audio.loop = true;
      audio.volume = 0.01;
      audio.setAttribute('playsinline', '');
      this.iosKeepAliveAudio = audio;
    }

    // Unconditionally attempt play() to capture the current user gesture
    this.iosKeepAliveAudio.play().catch(() => {});

    if (typeof navigator !== 'undefined' && 'mediaSession' in navigator) {
      try { navigator.mediaSession.playbackState = 'playing'; } catch (e) {}
    }

    // Setup visibility handler (only once)
    if (!this._visibilityHandler) {
       // ... keep existing _visibilityHandler logic ...
       document.addEventListener('visibilitychange', this._visibilityHandler);
    }
  } catch (e) {
    console.warn('iOS keepalive warning:', e);
  }
}

```
