# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The previous context injection bypassed Tone.js's native polyfill (`standardized-audio-context`), which successfully fixed the Worklet initialization but destroyed Tone.js's Web Worker clock initialization. As a result, the `Tone.Transport` timeline is frozen, leaving the generative loops (Layers 2 and 3) silent while the imperatively triggered drone (Layer 1) plays normally.

**Objective:** Revert the manual context injection to restore `Tone.Transport` functionality. Implement a `MediaStream` topological bridge in `BootEngine` to connect the Tone.js graph to the native `AudioWorkletNode` graph. This "air-gaps" the contexts, routing audio perfectly while bypassing all strict context-matching type errors.

## 2. Technical Decisions & Dependencies
* **MediaStream Bridge:** We will use `createMediaStreamDestination()` to extract the Tone.js output stream, and `createMediaStreamSource()` to inject it into the native hardware context where the Worklet resides.
* **Component Reversion:** `PlayButton.jsx` will be reverted to standard `await Tone.start()` to guarantee the internal Web Worker clock spins up correctly.

### Phase 2: Implement MediaStream Bridge (GenerativeAudioController.js) — ✅ Completed
* **Objective:** Connect the Tone.js graph to the native Worklet graph using a MediaStream buffer to prevent `InvalidAccessError`.
* **Instructions:**
    1. Open `src/engine/GenerativeAudioController.js`.
    2. Locate the `async bootEngine()` method.
    3. Replace the entire context extraction, Worklet initialization, and routing blocks (Steps 1 through 4) with this precise logic:
        ```javascript
        // 1. Create a Tone MediaStreamDestination as the Tone graph exit point
        const toneStreamDest = Tone.getContext().rawContext.createMediaStreamDestination();
        
        // 2. Instantiate Tone buses
        this.masterBus = new Tone.Volume(-6).toDestination();
        this.reverbInputBus = new Tone.Gain(1);
        
        // 3. Connect Tone graph to the stream destination
        this.reverbInputBus.connect(toneStreamDest);

        // 4. Extract pure native context for the Worklet
        const toneCtx = Tone.getContext();
        const rawCtx = toneCtx.rawContext || toneCtx._context || toneCtx;
        const nativeContext = rawCtx._nativeAudioContext || rawCtx._nativeContext || rawCtx._context || rawCtx;

        if (!nativeContext) {
          console.warn("Native context extraction failed. Falling back.");
        }

        // 5. Initialize Native AudioWorklet & MediaStream Bridge
        try {
          if (nativeContext && nativeContext.audioWorklet) {
            await nativeContext.audioWorklet.addModule('/assets/fdn-reverb-worklet.js');
            this.reverbWorklet = new AudioWorkletNode(nativeContext, 'fdn-reverb-worklet', {
              numberOfInputs: 1,
              numberOfOutputs: 1,
              outputChannelCount: [2],
            });
            this.reverbNode = this.reverbWorklet;

            // Bridge the Stream to the Native Context
            const nativeSource = nativeContext.createMediaStreamSource(toneStreamDest.stream);
            
            // Route native stream -> worklet -> native destination
            nativeSource.connect(this.reverbWorklet);
            const nativeGain = nativeContext.createGain();
            nativeGain.gain.value = 0.5; // -6dB headroom
            this.reverbWorklet.connect(nativeGain);
            nativeGain.connect(nativeContext.destination);
          } else {
            throw new Error("Worklet not supported on this context.");
          }
        } catch (e) {
          console.warn('Worklet load fallback/test environment:', e);
          this.reverbWorklet = null;
          // Fallback routing if bridge fails
          this.reverbInputBus.connect(this.masterBus);
        }
        ```
    4. Ensure the Layer 1, Layer 2, and Layer 3 Synth initializations (Step 5) remain exactly as they were, connecting to `this.reverbInputBus`.
